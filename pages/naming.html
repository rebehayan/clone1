Class Guide

BEM

BEM의 class명은 구체적이고 명료하여 HTML안에서도 읽기 쉬워야 하고 의미를 분명하게 전달해야 한다.

Block

재사용이 가능한 콘텐츠 영역을 Block이라 한다. 네임은 한 단어로 사용한다. 만약 네이밍이 길어질경우 케밥기법( - )을 이용한다. 
        <div class="login-wrap"></div>

Element

element는 블럭안에 특정 기능을 수행하는 컴포넌트를 말한다. element는 상황에 따라 달라지며 두개의 밑줄 __ 로 block다음에 작성한다. 
        <div class="login-wrap">
  <div class="login-wrap__header"></div>
</div>

Modifiers

block, element의 외관이나 상태를 변화시킬때 사용한다. 두개의 대쉬 --를 추가하여 표현한다. 
        <div class="login-wrap">
  <div class="login-wrap__header">
    <button class="**btn-login--active**"></button>
  </div>
</div>

Image Guide

  • 이미지 네이밍의 조합은 '형태_의미_순서_상태'의 순서로 조합한다.
  • 같은 이름의 이미지가 두 개 이상 존재하면 파일 언더스코어(_)로 구분한다.
  • 이미지 네이밍은 이미지의 확장자에 관계 없이 순차적으로 적용한다. 예) btn_confirm.gif, btn_confirm2.jpg, btn_confirm3.png
  • 임시이미지에 한하여 @특수문자를 처음으로 사용할 수 있다.
  • 이미지 예약어에 숫자, 영문소문자, 언더스코어(_)를 조합하여 사용할 수 있다.

Label Guide

Checkbox Label Naming

체크박스는 chk + 영역숫자 + 영역의갯수 숫자 순서로 기입한다.

              <input type="checkbox" id="chk1_1" /><label for="chk1_1">전체</label>
<input type="checkbox" id="chk1_2" /><label for="chk1_2">리모델링</label>
<input type="checkbox" id="chk1_3" /><label for="chk1_3">부엌</label>

<input type="checkbox" id="chk2_1" /><label for="chk2_1">선택1</label>
<input type="checkbox" id="chk2_2" /><label for="chk2_2">선택2</label>
<input type="checkbox" id="chk2_3" /><label for="chk2_3">선택3</label>

Radio Label Naming

라디오버튼은 ra + 영역숫자 + 영역의갯수 숫자 순서로 기입한다.

        <input type="radio" name="rag1" id="ra1_1" /><label for="ra1_1">선택1</label>
<input type="radio" name="rag1" id="ra1_2" /><label for="ra1_2">선택2</label>

<input type="radio" name="rag2" id="ra2_1" /><label for="ra2_1">선택1</label>
<input type="radio" name="rag2" id="ra2_2" /><label for="ra2_2">선택1</label>

Input Label Naming

input, textarea등 입력서식은 txt + 영역갯수 + 영역안의갯수 숫자 순서로 기입한다.

        <label for="txt1_1">설명</label><input type="text" id="txt1_1">
<label for="txt1_2">설명</label><input type="text" id="txt1_2">
        
<label for="txt2_1">설명</label><input type="text" id="txt2_1">
<label for="txt2_2">설명</label><input type="text" id="txt2_2">

Space Guide

  • 컴퍼넌트와 컴퍼넌트 사이의 공간이 필요할 때만 여백 클래스를 사용한다.
  • 여백은 한 방향으로만 제공해야 한다.
  • 여백의 크기는 디자이너와 협의후 정의한다.
