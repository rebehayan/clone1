HTML Guide

파일경로

작업이 완료된 HTML 파일은 로컬 브라우저에서 확인을 위해 상대경로로 저장한다.

        <link rel="stylesheet" type="text/css" href="./css/style.css">
<script type="text/javascript" src="./js/global.js"></script>
<img src="./images/img.svg" alt="대체텍스트">

head영역 정의

        <meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>페이지 제목</title>
<meta property="og:type" content="website">
<meta property="og:site_name" content="사이트명">
<meta property="og:title" content="페이지 제목">
<meta property="og:description" content="페이지 설명">
<meta property="og:image" content="/favicon.svg">
<meta property="og:url" content="./url">
<meta name="keywords" content="페이지 키워드">
<meta name="description" content="페이지 설명">
<link rel="stylesheet" href="./css/global.css">
<link rel="icon" href="/favicon.svg">
<script type="module" defer src="./js/global.js"></script>

표준 문법의 사용

  • HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • DTD를 제외한 모든 엘리먼트와 속성을 소문자로 작성한다.
  • 특수기호는 Entity name을 사용하여 entity 코드로 변환한다. Entity 코드는 ISO-8859-1을 기준으로 한다.

코드 들여쓰기

  • 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다.
  • 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓰고, 1탭의 크기는 공백 2칸으로 설정한다.

빈줄

  • 그룹을 감싸고 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것은 허용한다.
  • 빈 줄의 간격이 1줄을 초과하지 않으며 개발과 연동되는 그룹의 경우는 빈줄보다 주석을 작성한다.
<div class="side_content">
  content...
</div>

<div class="primary_content">

  <div class="content_section">
  content...
  </div>

</div>

주석

  • 코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 주석사용을 준수한다.
  • 빈 줄의 간격이 1줄을 초과하지 않으며 개발과 연동되는 그룹의 경우는 빈줄보다 주석을 작성한다.
  • 주석 기호와 주석 내용 사이에는 다른 기호가 올 수 없다.
  • 주석 기호와 주석 내용 사이에는 반드시 공백 한 칸이 있어야 한다.
  • 레이아웃 엘리먼트를 제외한 독립된 콘텐츠 영역의 주석 표기는 선택 사항이다.
